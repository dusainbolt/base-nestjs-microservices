# Báo cáo: Global Exception Handling trong Kiến trúc Microservices

*Ngày triển khai: 03-04-2026*

---

## 1. Vấn đề trước khi cải thiện

Trước đây, khi một microservice ném lỗi (ví dụ: `BadRequestException` với danh sách validation errors), thông tin chi tiết bị **mất dần** qua từng lớp xử lý:

```
ProductService throws BadRequestException({
  message: ['price must be a number', 'name should not be empty'],
  error: 'Bad Request'
})

↓ RmqInterceptor chỉ lấy err.message (string)
{ __isRpcError: true, statusCode: 400, message: "Bad Request Exception" }

↓ rpcToHttp tạo HttpException với string
new HttpException("Bad Request Exception", 400)

↓ Client nhận được response thiếu chi tiết
{ statusCode: 400, message: "Bad Request Exception" }
  ← Mất mảng validation errors!
  ← Không có timestamp, path
```

---

## 2. Giải pháp: 3 lớp xử lý lỗi cải tiến

### Lớp 1 — `RmqInterceptor` (Microservice side)

**File:** `libs/common/src/rmq/rmq.interceptor.ts`

Chặn lỗi tại microservice, giữ lại **toàn bộ** response body từ `HttpException.getResponse()`:

```typescript
// TRƯỚC: chỉ lấy message string
const message = err.message || 'Internal server error';
return of({ __isRpcError: true, statusCode, message });

// SAU: giữ full response object (bao gồm validation errors array)
const response = err instanceof HttpException ? err.getResponse() : null;
const errorBody = typeof response === 'object' ? response : { message: err.message };
return of({ __isRpcError: true, statusCode, ...errorBody });
```

**Kết quả truyền qua RabbitMQ:**
```json
{
  "__isRpcError": true,
  "statusCode": 400,
  "message": ["price must be a number", "name should not be empty"],
  "error": "Bad Request"
}
```

### Lớp 2 — `rpcToHttp()` (Gateway side — RxJS operator)

**File:** `libs/common/src/utils/rpc-to-http.util.ts`

Reconstruct `HttpException` với **full body** thay vì chỉ message string:

```typescript
// TRƯỚC
return throwError(() => new HttpException(message, status));

// SAU: giữ nguyên cấu trúc lỗi gốc
const { __isRpcError, statusCode, ...errorDetails } = err;
const responseBody = { statusCode: status, ...errorDetails };
return throwError(() => new HttpException(responseBody, status));
```

### Lớp 3 — `GlobalExceptionFilter` (Gateway side — Final output)

**File:** `libs/common/src/filters/global-exception.filter.ts`

Bắt **mọi** exception (HttpException, Error, unknown) và chuẩn hóa thành format thống nhất:

```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": ["price must be a number", "name should not be empty"],
  "timestamp": "2026-04-03T06:30:00.000Z",
  "path": "/products"
}
```

**Đăng ký:** `api-gateway.module.ts` → `{ provide: APP_FILTER, useClass: GlobalExceptionFilter }`

## 3. Luồng lỗi hoàn chỉnh (Error Flow)

```
┌──────────────────────┐
│   Microservice       │
│   throws Exception   │
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│   RmqInterceptor     │  ← Giữ full error body
│   catchError(err)    │  ← ACK (4xx) hoặc NACK→DLQ (5xx)
│   return of({...})   │  ← Trả object qua RabbitMQ
└──────────┬───────────┘
           ▼ (RabbitMQ)
┌──────────────────────┐
│   rpcToHttp()        │  ← Reconstruct HttpException
│   catchError(err)    │  ← Giữ nguyên message array
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│ GlobalExceptionFilter│  ← Format chuẩn + metadata
│   catch(exception)   │  ← Log (warn 4xx / error 5xx)
│   response.json()    │  ← Trả về client
└──────────────────────┘
```

---

## 4. Các trường hợp xử lý

| Loại lỗi | Ví dụ | Status | RabbitMQ | Log Level |
| :--- | :--- | :--- | :--- | :--- |
| Validation Error | `@IsEmail()` fail | 400 | ACK (xóa) | `WARN` |
| Auth Error | Token expired | 401 | ACK (xóa) | `WARN` |
| Not Found | Product không tồn tại | 404 | ACK (xóa) | `WARN` |
| Forbidden | Không có quyền xóa | 403 | ACK (xóa) | `WARN` |
| System Error | DB connection fail | 500 | NACK → DLQ | `ERROR` + stack |
| Unknown Error | Unhandled exception | 500 | NACK → DLQ | `ERROR` + stack |

## 5. Ví dụ Response thực tế

### Validation Error (400)
```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": [
    "price must be a number conforming to the specified constraints",
    "name should not be empty"
  ],
  "timestamp": "2026-04-03T06:30:00.000Z",
  "path": "/products"
}
```

### Not Found (404)
```json
{
  "statusCode": 404,
  "error": "Not Found",
  "message": "Product not found",
  "timestamp": "2026-04-03T06:30:00.000Z",
  "path": "/products/invalid-id"
}
```

### Unauthorized (401)
```json
{
  "statusCode": 401,
  "error": "Unauthorized",
  "message": "Access token is invalid or expired",
  "timestamp": "2026-04-03T06:30:00.000Z",
  "path": "/products"
}
```

### Internal Server Error (500)
```json
{
  "statusCode": 500,
  "error": "Internal Server Error",
  "message": "Internal server error",
  "timestamp": "2026-04-03T06:30:00.000Z",
  "path": "/products"
}
```
*Lưu ý: Chi tiết lỗi 500 chỉ được log trên server (kèm stack trace), không expose ra client.*
